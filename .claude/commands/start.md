Start a working session on the Baseline Fitness app:

1. Confirm you've loaded the project context (CLAUDE.md coding standards, security
   rules, Supabase table rules, and the architecture notes) and the memory index.
2. Start the live preview using the `fitness-app` configuration in
   `.claude/launch.json` (via the Claude Preview tools). Reuse the server if it's
   already running rather than starting a duplicate.
3. Take a screenshot to confirm the app is rendering.
4. If the app shows the sign-in screen, remind me to sign in myself — never enter
   my password. Once I confirm I'm signed in, screenshot again to verify the
   authenticated dashboard loaded.
5. Give me a one-line status (what's running, where) and ask what I want to work on.

Keep this brief — no long preamble. The goal is to get from a cold session to
"app is live, ready to work" in as few steps as possible.
